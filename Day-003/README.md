## 📖 Що вивчив
<!-- Опиши своїми словами що ти дізнався сьогодні -->
- розібрався зі складними типами даних: масивами (arrays) для списків, словниками (dictionaries) для збереження пар "ключ-значення", множинами (sets) для унікальних даних та перерахуваннями (enums) для створення власних категорій. 

## 💡 Ключові інсайти
<!-- Що тебе здивувало або що здалося особливо важливим? -->
- множини (set) працюють неймовірно швидко порівняно з масивами під час пошуку, бо не перевіряють кожен елемент по черзі. метод reversed() для масивів — це геніальна оптимізація: він не переписує дані задом наперед, а лише створює "віртуальну обгортку", щоб економити пам'ять комп'ютера. 

## 🤔 Що було складно
<!-- Чесно запиши що було незрозуміло — це допоможе відстежити прогрес -->
- величезний обсяг нової інформації! було складно одразу зрозуміти концепцію опціоналів у словниках (що значення за ключем може не існувати). довелося розбиратися, як використовувати значення за замовчуванням (default), щоб не було конфліктів типів даних (string та int).

## 💻 Код дня
<!-- Вставити найцікавіший фрагмент коду або посилання на файл -->
```swift
var inventory = ["Зілля здоров'я": 5, "Кристали мани": 10]
inventory["Слиз слайма"] = 3
inventory["Зілля здоров'я", default: 0] -= 1
print("У героїні залишилося \(inventory["Зілля здоров'я", default: 0]) лікувальних зіль")
```

## 📝 Нотатки з обговорення з AI
<!-- Короткий підсумок обговорення -->
- розібрали, як працює default: 0 під капотом — це спосіб дати системі "безпечне" значення на випадок пустоти, щоб можна було робити математичні операції або виводити чистий текст без Optional(...). розробили повноцінну систему ігрового інвентарю на базі словників!
