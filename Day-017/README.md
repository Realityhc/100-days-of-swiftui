## 💡 Ключові інсайти
<!-- Що тебе здивувало або що здалося особливо важливим? -->
-   декларативний дизайн — це магія! ми кажемо SwiftUI що хочемо отримати (наприклад, меню або сегментований список), а не пояснюємо, як це намалювати. система сама підбирає ідеальний вигляд.
-   обчислювальні властивості (var name: Type { ... }) нічого не зберігають у пам'яті. вони відслідковують зміни у @State, миттєво роблять розрахунки, віддають результат через return і одразу "забувають" його. це супер-ефективно!
-   на цифровій клавіатурі .decimalPad в iOS немає вбудованої кнопки "Готово". ми маємо створювати її власноруч, керуючи фокусом екрана через amountIsFocused = false.
-   у циклі ForEach параметр id: \.self працює як "роздача бейджів": він використовує саме значення елемента (наприклад, число 20) як його унікальний ідентифікатор.

## 📝 Нотатки з обговорення з AI
<!-- Короткий підсумок обговорення -->
-   текст "Amount" у TextField — це не просто коробка для цифр, це етикетка/плейсхолдер. якщо поле має значення за замовчуванням (наприклад 0.0), текст ховається, але залишається в коді для систем доступності (VoiceOver), щоб озвучувати екран незрячим людям.
-   модифікатор format: .percent відповідає ВИКЛЮЧНО за красу і значок % на екрані. під капотом для Swift це все ще звичайні цілі числа, які для розрахунків потрібно вручну ділити на 100.
-   ідеальне місце для .navigationTitle — всередині NavigationStack, безпосередньо на самій Form. NavigationStack — це порожня рамка, а Form — це картина, і табличка з назвою має висіти саме на картині!
